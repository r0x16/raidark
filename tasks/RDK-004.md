# RDK-004 — CORS configurable y CSRF apagable por servicio.

## Ubicación
- **Repositorio:** Raidark (`github.com/r0x16/Raidark`)
- **Componente:** `shared/api/driver/EchoApiProvider`
- **Épica técnica:** `EP-RDK-HTTP` — Capa HTTP base de Raidark

## Tarea técnica
- **Tipo:** DEVELOPMENT
- **Estado:** Completed
- **Quién:** DEV
- **Qué:** Permitir que cada servicio decida si activa o no el middleware CSRF builtin de Raidark, y que configure CORS por variable de entorno con lista de orígenes.
- **Cómo:**
  - Confirmar y formalizar que `CSRF_ENABLED=false` apaga completamente el middleware CSRF y no monta el endpoint `/csrf-token` cuando está deshabilitado.
  - `CORS_ALLOW_ORIGINS` acepta lista separada por coma; orígenes vacíos rechazados.
  - Soporte para `CORS_ALLOW_HEADERS`, `CORS_ALLOW_METHODS`, `CORS_ALLOW_CREDENTIALS`, `CORS_MAX_AGE`.
  - Defaults sanos: si `CORS_ALLOW_ORIGINS` no está definido, no se monta middleware CORS (decisión explícita en lugar de wildcard).
  - Logging del estado de los toggles en bootstrap (`csrf=disabled`, `cors=https://a.example, https://b.example`).
- **Por qué:** Servicios que viven detrás de un BFF que ya implementa CSRF no deben sumar otra capa contradictoria. Servicios que sirven a clientes browser directamente sí. La configuración debe ser explícita por servicio.
- **Cuándo:** Sin bloqueantes.

## Variables de entorno
```
CSRF_ENABLED=true|false
CORS_ALLOW_ORIGINS=https://a.example,https://b.example
CORS_ALLOW_HEADERS=Content-Type,Authorization,X-Correlation-ID
CORS_ALLOW_METHODS=GET,POST,PUT,PATCH,DELETE,OPTIONS
CORS_ALLOW_CREDENTIALS=true
CORS_MAX_AGE=600
```

## Criterio de aceptación
- Tests de integración con Echo:
  - `CSRF_ENABLED=false` → no hay endpoint `/csrf-token` y POST sin token pasa.
  - `CSRF_ENABLED=true` → POST sin token recibe 403 con envelope estándar (`RDK-002`).
  - `CORS_ALLOW_ORIGINS` no definido → preflight responde 404/sin headers CORS.
  - Lista por coma se parsea correctamente.
- Documentado en `docs/configuration/cors.md` y `docs/configuration/csrf.md`.

## Fuera de alcance
- CSRF basado en double-submit cookie (la implementación actual usa header token).
- Per-route overrides de CORS.

## Historia de usuario relacionada
- **Actor:** Equipo desarrollador que despliega servicios sobre Raidark con distintas topologías.
- **Historia:** Como desarrollador desplegando servicios sobre Raidark, quiero apagar CSRF cuando mi servicio vive detrás de un BFF y configurar CORS por env, para no duplicar mecanismos de protección y para que la lista de orígenes permitidos sea declarativa y revisable.
- **Valor esperado:** Cada servicio tiene una sola fuente de verdad sobre su política CSRF/CORS, expresada en `.env` y validada en bootstrap.

**Contexto previo encontrado:**
La implementación parcial ya existía en `shared/api/driver/EchoApiProvider.go`:

- CORS ya leía variables de entorno pero usaba `["*"]` como default (wildcard peligroso) y no tenía `CORS_MAX_AGE`.

**Cambios implementados:**

- Import `"strings"` agregado para `strings.Join` en el log.

- El endpoint `/csrf-token` ahora **solo se registra** cuando `CSRF_ENABLED=true`. Antes se registraba siempre y hacía el check en runtime.

- Agregada variable `CORS_MAX_AGE=600`.

### 2026-05-02 — cierre

**Resultado final consolidado:**

- CORS: middleware montado solo cuando `CORS_ALLOW_ORIGINS` está explícitamente definido en el entorno (opt-out seguro, sin wildcard por defecto). Filtra orígenes vacíos. Soporta `CORS_ALLOW_HEADERS`, `CORS_ALLOW_METHODS`, `CORS_ALLOW_CREDENTIALS`, `CORS_MAX_AGE`. Boot log con `cors=<lista>` o `cors=disabled`.
- CSRF: `CSRF_ENABLED=false` (default) omite middleware y no registra la ruta `/csrf-token`. `CSRF_ENABLED=true` monta el middleware y registra la ruta. Boot log con `csrf=enabled` o `csrf=disabled`. Handler del endpoint extraído a `csrfTokenAction` (método separado).
- Documentación creada: `docs/configuration/cors.md` y `docs/configuration/csrf.md`.
- `.env-example` actualizado con `CORS_MAX_AGE=600`.

**Archivos finales:**
- `shared/api/driver/EchoApiProvider.go`
- `shared/api/driver/modules/EchoMainModule.go`
- `.env-example`
- `docs/configuration/cors.md`
- `docs/configuration/csrf.md`

**Decisiones de diseño:**
- CORS opt-out en lugar de wildcard: seguridad explícita sobre conveniencia.
- `/csrf-token` no registrado cuando disabled: cero superficie cuando el feature está apagado.
- `csrfTokenAction` separado de `Setup()`: separación entre registro y procedimiento.
