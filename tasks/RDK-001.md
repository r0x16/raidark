# RDK-001 — Helper UUIDv7 para identificadores ordenables por tiempo.

## Ubicación
- **Repositorio:** Raidark (`github.com/r0x16/Raidark`)
- **Componente:** `shared/ids`
- **Épica técnica:** `EP-RDK-CORE` — Primitivas básicas del framework

## Tarea técnica
- **Tipo:** DEVELOPMENT
- **Estado:** In Progress
- **Quién:** DEV
- **Qué:** Proveer un paquete `shared/ids` que genere y valide UUIDv7 según RFC 9562, utilizable como PK de modelos GORM y como identificador para mensajes, payloads y agregados.
- **Cómo:**
  - Función `ids.NewV7() string` que devuelva un UUIDv7 codificado canónicamente (8-4-4-4-12 hex con guiones).
  - Función `ids.IsValidV7(s string) bool` que valide forma y bits de versión/variante.
  - Custom GORM data type `ids.UUIDv7` (o helper de hook) para usar como PK con generación automática en `BeforeCreate` cuando el campo está vacío.
  - Implementación basada en `github.com/google/uuid` v1.6+ que ya soporta `NewV7`, sin dependencias extra.
- **Por qué:** Distintos servicios y librerías inventarían generadores distintos (UUIDv4, ULIDs ad hoc, autoincrement). UUIDv7 ordenable por tiempo es un piso técnico común que mejora el rendimiento de índices BTree y simplifica debugging cronológico.
- **Cuándo:** Sin bloqueantes. Es habilitante para el resto del catálogo (eventos, outbox, idempotency keys, models con PK).

## Criterio de aceptación
- Paquete `shared/ids` exportable.
- Tests unitarios cubren: generación válida, monotonicidad lógica entre dos llamadas consecutivas (timestamp embebido no decrece), `IsValidV7` rechaza UUIDv4 y strings malformados.
- Test de integración GORM (sqlite in-memory) que crea y lee un registro con PK `ids.UUIDv7` autogenerado.
- Documentado en `docs/ids/uuidv7.md` con ejemplo de uso como PK GORM y como identificador suelto.

## Fuera de alcance
- ULIDs u otros formatos alternativos.
- Migraciones para reemplazar UUIDv4 existentes en bases ya desplegadas (responsabilidad del consumidor).

## Historia de usuario relacionada
- **Actor:** Equipo desarrollador que construye servicios sobre Raidark.
- **Historia:** Como desarrollador construyendo un servicio sobre Raidark, quiero un helper estándar para generar UUIDv7 ordenables por tiempo, para no inventar generadores propios y para que mis PKs e identificadores de mensajes mantengan orden temporal natural.
- **Valor esperado:** Todos los servicios sobre Raidark comparten un mismo formato de ID, los índices BTree de PKs se mantienen densos y el orden cronológico se infiere del propio ID.
