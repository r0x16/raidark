# RDK-005 — Storage adapter abstracto con driver filesystem.

## Ubicación
- **Repositorio:** Raidark (`github.com/r0x16/Raidark`)
- **Componente:** `shared/storage`
- **Épica técnica:** `EP-RDK-STORAGE` — Almacenamiento de objetos

## Tarea técnica
- **Tipo:** DEVELOPMENT
- **Estado:** In Progress
- **Quién:** DEV
- **Qué:** Definir una abstracción `StorageProvider` para almacenar objetos binarios (imágenes, PDFs, otros archivos) con visibilidad pública o privada y URLs firmadas, y entregar un driver `filesystem` v1 con HMAC para signed URLs.
- **Cómo:**
  - Interfaz Go:
    ```go
    type StorageProvider interface {
        Put(ctx context.Context, key string, r io.Reader, opts PutOptions) (PutResult, error)
        Get(ctx context.Context, key string) (io.ReadCloser, ObjectInfo, error)
        Delete(ctx context.Context, key string) error
        SignedURL(ctx context.Context, key string, ttl time.Duration) (string, error)
        PublicURL(key string) string
        Exists(ctx context.Context, key string) (bool, error)
    }
    type Visibility int
    const (
        VisibilityPublic Visibility = iota
        VisibilityPrivate
    )
    type PutOptions struct {
        Visibility  Visibility
        ContentType string
        Size        int64
    }
    type PutResult struct {
        Key       string
        SizeBytes int64
        ETag      string
    }
    type ObjectInfo struct {
        Key         string
        SizeBytes   int64
        ContentType string
        ModifiedAt  time.Time
    }
    ```
  - Driver `filesystem`:
    - Dos raíces: `STORAGE_PUBLIC_ROOT` y `STORAGE_PRIVATE_ROOT`.
    - `Put` recibe `io.Reader` y escribe streaming (sin buffer completo en memoria).
    - `SignedURL` para objetos `Private` se emula con HMAC firmado sobre `(key, expires_at)`; el handler estático interno expuesto por Raidark valida firma y TTL antes de servir.
    - `PublicURL` concatena `STORAGE_PUBLIC_BASE_URL + key`.
  - Convención obligatoria de keys (validada en helper, no impuesta en interfaz): `{namespace}/{usage}/{año}/{mes}/{uuid}{ext}`. El consumidor decide `namespace` y `usage`.
  - Provider factory `StorageProviderFactory` registrable en el provider hub.
- **Por qué:** Sin abstracción, cada servicio guarda archivos en su propia carpeta con nombres impredecibles. Una interfaz común permite migrar a S3/MinIO/GCS más adelante sin cambiar código de aplicación.
- **Cuándo:** Bloqueante recomendado: `RDK-001` (UUIDv7 para keys).

## Variables de entorno
```
STORAGE_DRIVER=filesystem
STORAGE_PUBLIC_ROOT=/storage/public
STORAGE_PRIVATE_ROOT=/storage/private
STORAGE_PUBLIC_BASE_URL=https://cdn.example.com
STORAGE_SIGNING_SECRET=<hex>
STORAGE_SIGNED_URL_DEFAULT_TTL=600s
```

## Criterio de aceptación
- Paquete `shared/storage` con interfaz, tipos y driver `filesystem`.
- Handler interno para servir objetos privados con verificación HMAC y TTL.
- Tests:
  - Round-trip Put/Get con `io.Reader` que no carga todo en memoria (test con tamaño > 10MB).
  - `SignedURL` válida dentro de TTL, expira cuando vence.
  - `SignedURL` rechazada si el HMAC no coincide.
  - `Delete` borra archivo y `Exists` devuelve false.
- Documentado en `docs/storage/adapter.md` y `docs/storage/filesystem-driver.md`.

## Fuera de alcance
- Driver S3 / MinIO / GCS (queda para tarea futura siguiendo la misma interfaz).
- Replicación / backups.
- Quotas por consumidor.

## Historia de usuario relacionada
- **Actor:** Equipo desarrollador que necesita persistir archivos binarios en un servicio.
- **Historia:** Como desarrollador construyendo un servicio sobre Raidark, quiero una interfaz de storage estándar con visibilidad pública/privada y URLs firmadas, para no acoplarme al filesystem desde el día uno y poder migrar a S3 cambiando solo configuración.
- **Valor esperado:** El código de aplicación habla con `StorageProvider`; cambiar de driver es un cambio de provider en bootstrap.

## Bitácora make

### 2026-05-02 — sesión 1

- Created `shared/storage/domain/StorageProvider.go`: `StorageProvider` interface, `Visibility`, `PutOptions`, `PutResult`, `ObjectInfo` types.
- Created `shared/storage/domain/KeyHelper.go`: `ValidateKey(key string) error` and `BuildKey(namespace, usage, ext string) (string, error)`. Validates 4-segment format, rejects empty/absolute/traversal keys, checks UUID stem with `ids.IsValidV7`.
- Created `shared/storage/driver/FilesystemStorageProvider.go`: full filesystem driver with streaming `Put` (TeeReader + MD5 ETag), two-root `Get`/`Exists`/`Delete`, idempotent `Delete`, `PublicURL`, `SignedURL` with HMAC-SHA256 over `"{key}\n{expires_unix}"`. Path traversal guard on every root join.
- Created `shared/storage/driver/FilesystemSignedUrlHandler.go`: `NewSignedUrlHandler` — validates `expires` timestamp, constant-time HMAC comparison, serves private file via `http.ServeContent` (Range/ETag/304 support). Returns 403/404 with `rest.RESTError` on failure.
- Created `shared/api/driver/modules/EchoStorageModule.go`: unconditionally registered module that short-circuits when no `StorageProvider` is in the hub or when the active driver is not `*FilesystemStorageProvider`. Registers `GET /_storage/*`.
- Created `shared/providers/driver/StorageProviderFactory.go`: factory reads `STORAGE_DRIVER` (default `filesystem`), constructs driver, registers `StorageProvider` interface type in the hub.
- Modified `raidark.go`: registered `EchoStorageModule` after `EchoMetricsModule` in `registerModules()`.
- Created `docs/storage/adapter.md` and `docs/storage/filesystem-driver.md`.
- `go build ./...` passes with no errors.
